# EBSi-Download-and-Management

EBSi 기출문제 페이지에서 수능, 모의고사, 학력평가 기출문제를 일괄 다운로드하고, 글/그림/표 에셋도 따놓으세요!
-박승수

## 눈에 띄는 기능

- 평가원/교육청을 나누어 과목을 선택하고, 받을 수 있는 자료를 표로 확인합니다.
- 국어, 수학, 영어, 한국사, 사회탐구, 과학탐구, 직업탐구, 제2외국어/한문 과목의 EBSi 과목 ID를 기본 제공합니다.
- 평가원 자료는 6월/9월/수능뿐 아니라 EBSi 월별 목록 전체를 확인하고, 공식 제목을 기준으로 모의평가/수능을 다시 분류합니다.
- 교육청 자료는 3월, 4월, 5월, 7월, 10월 학력평가 중심으로 확인합니다.
- 문제 PDF, 빠른 정답 이미지, 해설 PDF를 선택 다운로드합니다. 영어 과목은 듣기 MP3와 대본 PDF도 함께 확인합니다.
- 다운로드 원본은 `downloads/<과목>/current`에 보관하고, 사용자가 지정한 저장 경로로 복사할 수 있습니다.
- 별도 하위 모듈에서 문제/해설 PDF를 Markdown, HTML, DOCX로 전사 및 재조립합니다.

## Disclaimer

이 스크립트는 오로지 교사와 학생의 더욱 편한 학습 및 자료 관리를 돕기 위한 도구입니다. 배포자는 대한민국 평가원과 대한민국 교육청의 저작권을 존중하며, 실제 시험 문제, 빠른 정답 이미지, 해설 자료를 이 저장소에 탑재하거나 재배포하지 않습니다.

## 저장소 원칙

실제 시험 자료는 GitHub에 올리지 않습니다. 이 저장소에는 코드, 설정 예시, 빈 출력 폴더 구조, 그리고 "이 위치에 이런 파일이 생성된다"는 설명만 둡니다.

다운로드 결과 폴더에는 보통 아래 파일이 들어갑니다.

- 문제 PDF
- 빠른 정답 이미지
- 해설 PDF
- 영어 과목의 경우 듣기 MP3와 대본 PDF

영어 듣기 음성은 보관 대상일 뿐, 이 도구가 음성을 자동 전사하지는 않습니다. 제공되는 대본은 별도 자료로 함께 관리합니다.

## 빠른 사용 순서

1. `KICE_시험자료_관리_UI.vbs`를 실행합니다.
2. `시험 구분`에서 평가원 또는 교육청을 고릅니다.
3. `과목`을 선택합니다.
4. `EBS 현황 확인`을 누릅니다.
5. 표에서 필요한 문제/정답/해설 행을 선택합니다.
6. `선택 다운로드`를 누릅니다.
7. `선택 항목 저장`으로 원하는 저장 경로에 복사합니다.
8. 필요하면 `다운로드 원본 전사/재조립`을 눌러 다운로드 시험 폴더 안에 HTML/DOCX 산출물을 만듭니다.

## 버튼 사용법

### 핵심 흐름

- `EBS 현황 확인`: 선택한 시험 구분/과목에서 받을 수 있는 문제, 빠른 정답, 해설을 표에 표시합니다. 영어 과목은 듣기와 대본도 함께 표시합니다.
- `선택 다운로드`: 표에서 선택한 가능 항목만 다운로드 원본 폴더에 저장합니다.
- `선택 항목 저장`: 다운로드 원본 폴더의 파일을 사용자가 지정한 저장 경로로 복사합니다.
- `저장 경로 열기`: 복사 결과를 확인할 폴더를 엽니다.

### EBS 목록

- `가능 항목 전체 선택`: 현재 다운로드 가능한 행을 한 번에 선택합니다.
- `선택 해제`: 표 선택을 모두 지웁니다.
- 표 머리글을 누르면 학년도, 회차, 문서 종류, 상태, 공식 제목 기준으로 정렬합니다.
- 여러 행은 `Ctrl` 또는 `Shift`로 선택할 수 있습니다.

### 전사 및 재조립

- `다운로드 원본 전사/재조립`: 다운로드한 문제/해설 PDF가 있는 시험 폴더 아래에 `6월 전사 및 조립`, `수능 전사 및 조립` 같은 하위 폴더를 만들고 Markdown, HTML, DOCX로 변환합니다.
- `재조립 결과 열기`: 과목 다운로드 원본 폴더를 엽니다.
- `재조립 로그 열기`: 전사/재조립 과정의 로그를 엽니다.

DOCX 산출물은 그림/표 아래에 검토용 추출 텍스트를 붙입니다. 복잡한 표, 말풍선, 스캔본은 로컬 코드만으로 완벽하지 않을 수 있으므로 AI Agent 환경에서 재검수하는 흐름을 권장합니다.

### 편의 기능

- `환경 점검`: Python 실행 환경, EBSi 접속, 기본 폴더 상태를 확인합니다.
- `관리 대상 폴더 확인`: 사용자가 자료를 모아 둔 폴더를 엽니다.
- `관리 대상 점검`: 관리 대상 폴더를 스캔해 보유 현황 보고서를 만듭니다.
- `공식 다운로드 새로고침`: 선택 과목의 전체 범위를 다시 확인하고 가능한 파일을 일괄 다운로드합니다.
- `누락/재다운로드 보고서`: 현재 보유분과 공식 원본을 대조해 보강이 필요한 항목을 찾습니다.
- `누락 보고서 기준 반영`: 누락 보고서에 잡힌 파일을 지정한 저장 경로로 복사합니다.
- `보고서 확인`: `reports` 폴더를 엽니다.
- `버튼 기능 안내`: 화면 안에서 각 버튼의 역할을 다시 확인합니다.

각 버튼의 세부 기능은 추후 패치에서 더 보강할 예정입니다.

## AI Agent 환경과 로컬 코드

### AI Agent 환경

- Codex는 `AGENTS.md`, Claude Code/Cowork는 `CLAUDE.md`, Google/Antigravity 계열은 `GEMINI.md` 또는 `ANTIGRAVITY.md`를 함께 컨텍스트로 두고 작업합니다.
- 문제 PDF, 해설 PDF, 영어 대본을 함께 보고 문항 단위로 재조립하는 검수에 적합합니다.
- 그림, 표 내부 글자, 말풍선처럼 로컬 추출이 흔들릴 수 있는 영역을 다시 읽는 데 유리합니다.
- 처리 로그와 메타데이터를 근거로 남기며, 최종 검수본을 만드는 용도로 사용합니다.

### 로컬 코드

- EBSi 목록 확인, 선택 다운로드, 저장 경로 복사, 기본 PDF 전사/재조립을 자동화합니다.
- 인터넷 접속과 로컬 Python 환경이 필요합니다.
- 영어 듣기 음성 전사는 하지 않습니다.
- 전사 결과는 검토용 초안으로 보고, 실제 수업/배포 자료로 쓰기 전에는 사람이 확인해야 합니다.

## 폴더 구조

```text
downloads/
  <과목>/
    current/
    legacy/
      YYYYMMDD-HHMMSS/

reports/
  *.csv
  *.json

test_crawl_and_reassemble/
  intermediate/
  output/
  logs/
```

- `current`: 최신 공식 다운로드 원본
- `legacy`: 같은 파일명인데 내용이 바뀐 이전 원본
- `reports`: 현황, 다운로드, 복사, 누락 보고서
- `test_crawl_and_reassemble/output`: 전사/재조립 결과
- `test_crawl_and_reassemble/logs`: 전사/재조립 로그
- `downloads/<과목>/current/<연도>/<회차>/<회차> 전사 및 조립`: GUI에서 실행한 전사/재조립 결과

`downloads`, `reports`의 상세 CSV/JSON, 전사 결과물은 로컬 산출물로 취급하며 GitHub에는 올리지 않습니다.

## 전사 및 재조립 산출물

`test_crawl_and_reassemble/`은 본편 다운로더와 분리된 하위 모듈입니다.

- Markdown 전사본
- 표/그림/스캔 이미지 에셋
- 레이아웃 JSON
- LLM 전달용 manifest
- HTML 재조립본
- DOCX 재조립본
- 처리 로그와 AI metadata

문제 PDF는 전사 대상으로 처리하고, 같은 시험의 빠른 정답 이미지와 해설 PDF는 `metadata.json`, `llm_manifest.json`, `llm_manifest.md`, `*.llm_bundle.md`에 공식 출처 메타데이터로 연결합니다.

## 베타 메모

- 현재는 베타 단계입니다.
- 실제 시험 파일은 포함하지 않습니다.
- 버튼별 상세 기능과 문항 단위 재조립 정확도는 계속 패치할 예정입니다.
- 2028학년도 통합사회/통합과학 체제 이후 관리 범위는 EBSi 공식 과목 ID가 안정되면 확장합니다.
