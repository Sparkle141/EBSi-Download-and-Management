# EBSi-Download-and-Management

EBSi 홈페이지에서 대한민국 평가원/교육청 사회탐구 모의고사/학력평가를 내려받고 정리하기 위한 관리 도구입니다.  
-박승수

## 주요 기능

- EBSi 공식 기출 목록에서 사회탐구 시험지를 확인합니다.
- 과목, 학년도, 회차, 문서 종류별로 문제 PDF, 빠른 정답 이미지, 해설 PDF를 내려받습니다.
- 이미 받은 파일은 해시로 비교해 중복 저장을 줄이고, 바뀐 파일은 `legacy`로 이관합니다.
- 현재 보유한 자료 폴더를 스캔해 현황, 중복 후보, 누락/재다운로드 가능 항목을 보고서로 만듭니다.
- 선택한 시험지만 다운로드하고, 다운로드 원본을 별도 저장 경로에 복사할 수 있습니다.
- 별도 하위 모듈에서 시험지 PDF를 텍스트/표/그림으로 전사하고 HTML/DOCX로 재조립할 수 있습니다.

## Disclaimer

이 스크립트는 오로지 교사와 학생의 더욱 편한 학습 및 자료 관리를 돕기 위한 도구입니다.
배포자는 대한민국 평가원과 대한민국 교육청의 저작권을 존중하며, 실제 시험 문제, 빠른 정답 이미지, 해설 자료를 이 저장소에 탑재하거나 재배포하지 않습니다.

## 공개 저장소 원칙

실제 시험 자료는 저작권 문제가 있을 수 있으므로 GitHub에 탑재하지 않습니다.
이 저장소에는 코드, 설정 예시, 빈 출력 폴더 구조, 집계 요약만 남깁니다.

- `downloads/`: 일반 다운로드 결과물이 생성되는 위치
- `official_downloads/`: 공식 출처 다운로드 결과물이 생성되는 위치
- `reports/summary.json`: 로컬 보관함의 집계 요약
- `reports/*.csv`, 상세 manifest, inventory: 로컬 경로와 대량 목록이 포함될 수 있어 제외
- `test_crawl_and_reassemble/output/`: 전사/재조립 테스트 산출물이 생성되는 위치이며 Git에는 올리지 않음

다운로드 결과 폴더에는 보통 아래 세 종류의 파일이 들어갑니다.

- 문제 PDF
- 빠른 정답 이미지
- 해설 PDF

GitHub에는 위 파일 자체를 올리지 않고, "이 위치에 이런 파일이 생성된다"는 설명만 유지합니다.

## 빠른 사용 흐름

1. `KICE_시험자료_관리_UI.vbs`를 실행합니다.
2. 시험 구분과 과목을 선택합니다.
3. `EBS 현황 확인`으로 현재 받을 수 있는 시험지를 확인합니다.
4. 표에서 필요한 문제/정답/해설 행을 선택합니다.
5. `선택 다운로드`로 필요한 항목만 내려받습니다.
6. 필요하면 `선택 항목 저장`으로 지정한 저장 경로에 복사합니다.
7. PDF를 전사하려면 `test_crawl_and_reassemble` 모듈에서 문제 PDF를 HTML/DOCX로 재조립합니다.

## UI 버튼 사용법

### 기본 경로

- `시험 구분`: 평가원/EBSi 또는 교육청 등 시험 출처를 고릅니다.
- `과목`: 생활과 윤리, 윤리와 사상 등 관리할 과목을 고릅니다.
- `관리 대상`: 사용자가 실제로 자료를 모아 둔 폴더입니다.
- `저장 경로`: 다운로드 원본을 최종 정리 폴더로 복사할 위치입니다.
- `다운로드 원본`: 공식 사이트에서 받은 최신 원본을 보관하는 `downloads/<과목>/current` 계열 위치입니다.
- `다음 학년도까지 확인`: 평가원 시험 기준으로 현재 연도 다음 학년도까지 확인합니다.

### 다운로드 전

- `환경 점검`: Python 실행 환경, EBSi 접속, 원본 폴더, 다운로드 폴더를 확인합니다.
- `관리 대상 점검`: 관리 대상 폴더를 스캔해 보유 현황 보고서를 만듭니다.
- `EBS 현황 확인`: EBSi에서 현재 받을 수 있는 공식 항목을 표에 표시합니다.

### EBS 목록

- 표에는 `학년도`, `회차`, `문서`, `상태`, `공식 제목`, `파일명`이 표시됩니다.
- 여러 행은 `Ctrl` 또는 `Shift`로 선택할 수 있습니다.
- 표 머리글을 누르면 해당 열 기준으로 정렬됩니다.
- `가능 항목 전체 선택`: 현재 다운로드 가능한 행을 한 번에 선택합니다.
- `선택 해제`: 표 선택을 모두 지웁니다.

### 다운로드 관련

- `선택 다운로드`: 표에서 선택한 가능 항목만 다운로드합니다.
- `선택 항목 저장`: `current` 다운로드 원본을 지정한 저장 경로에 복사합니다.
- `누락/재다운로드 보고서`: 현재 보유분과 공식 원본을 대조해 보강이 필요한 항목을 찾습니다.
- `공식 다운로드 새로고침`: 선택 과목의 전체 범위를 다시 확인하고 받을 수 있는 파일을 갱신합니다.
- `버튼 기능 안내`: 화면 안에서 각 버튼의 역할을 다시 확인합니다.

### 폴더와 보고서

- `보고서 확인`: `reports` 폴더를 엽니다.
- `저장 경로 열기`: 최종 복사 대상 폴더를 엽니다.
- `원본 폴더 확인`: 관리 대상 폴더를 엽니다.

각 버튼의 세부 동작은 추후 패치에서 더 자세히 보강할 예정입니다.

## 전사 및 재조립 모듈

`test_crawl_and_reassemble/`은 본편과 분리된 하위 모듈입니다.

이 모듈은 PDF 시험지를 다음 산출물로 변환합니다.

- Markdown 전사본
- 표/그림/스캔 이미지 에셋
- 레이아웃 JSON
- LLM 전달용 manifest
- HTML 재조립본
- DOCX 재조립본

EBSi 다운로더의 `reports/selected_download_sources_*.csv`, `reports/official_sources_*.csv`, `reports/official_exam_sources*.csv`, `reports/availability_*.csv`를 인식합니다.
문제 PDF는 전사 대상으로 처리하고, 같은 시험의 빠른 정답 이미지와 해설 PDF는 `metadata.json`, `llm_manifest.json`, `llm_manifest.md`, `*.llm_bundle.md`에 공식 출처 메타데이터로 연결합니다.

## 다운로드와 산출물 보관 정책

다운로드 원본은 과목별로 나뉩니다.

```text
downloads/
  생활과 윤리/
    current/
    legacy/
      YYYYMMDD-HHMMSS/
```

- `current`: 최신 공식 다운로드 원본
- `legacy`: 같은 파일명인데 내용이 바뀐 이전 원본
- 같은 파일을 다시 받았고 해시가 같으면 새 파일을 만들지 않습니다.
- 같은 파일명인데 내용이 다르면 기존 파일을 `legacy`로 옮기고 새 파일을 `current`에 둡니다.
