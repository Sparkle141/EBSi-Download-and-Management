# 환경 점검과 라우팅 설명

## 환경 점검에서 확인하는 것

`환경 점검` 버튼은 다음을 확인한다.

- Python 실행 환경이 있는지
- 원본 저장 폴더가 존재하는지
- 저장 경로, 즉 복사 목적지의 폴더 또는 부모 폴더가 존재하는지
- 보고서 폴더가 존재하거나 만들 수 있는지
- 다운로드 원본 폴더가 존재하거나 만들 수 있는지
- legacy 이관 폴더가 존재하거나 만들 수 있는지
- `www.ebsi.co.kr` DNS 조회가 되는지
- `wdown.ebsi.co.kr` DNS 조회가 되는지
- EBSi 기출문제 페이지가 HTTP 200으로 열리는지
- EBSi 기출 AJAX가 샘플 조건으로 응답하는지

현재 샘플 AJAX 조건은 고3 EBSi 기출, 사회탐구, 생활과 윤리, 2025년 6월 시행 시험이다. 이 점검은 “EBSi 다운로드 루트가 작동하는가”를 보는 용도이며, 모든 과목/모든 교육청 루트를 각각 검사하는 것은 아니다.

## 평가원/EBSi 과목을 바꿀 때

현재 구현된 평가원 쪽은 EBSi 고3 기출문제 루트를 사용한다.

기본 페이지:

```text
https://www.ebsi.co.kr/ebs/xip/xipc/previousPaperList.ebs?targetCd=D300
```

실제 조회는 같은 페이지의 AJAX 엔드포인트로 들어가며, 과목을 바꾸면 요청값이 바뀐다.

- `targetCd`: 고3 기출 대상 코드
- `arOrd`: 영역 코드. 사회탐구는 `5`
- `subjIdList`: 과목 ID. 생활과 윤리 `63002`, 윤리와 사상 `63003`
- `yearList`: 시행 연도. 평가원 학년도에서 1을 뺀 값
- `monthList`: 시행 월. 6월, 9월, 수능 등

따라서 생활과 윤리에서 윤리와 사상으로 바꿀 때는 “다른 EBSi 페이지를 검색”하는 것이 아니라, 같은 EBSi 기출 AJAX에 다른 `subjIdList`를 보내는 방식이다.

과목을 바꾸면 UI는 다음도 같이 바꾼다.

- 표시 과목명
- EBSi 과목 ID
- 다운로드 원본 폴더: `downloads/<과목>/current`
- legacy 폴더: `downloads/<과목>/legacy`
- 보고서 파일명
- 과목별 기본 원본 저장 폴더
- 과목별 기본 저장 경로, 즉 복사 목적지

## 교육청 선택 시

교육청 어댑터는 실험 기능으로 추가했다. 현재는 EBSi 고3 기출 AJAX에서 잡히는 학평 항목을 우선 사용한다.

평가원과 다른 기준:

- 공식 제목과 시행연도 중심
- 교육청 과목/학년/시행월 체계 보존
- 다운로드 원본 보관과 저장 경로 복사는 공통 정책 사용

현재 확인된 방식:

- 같은 EBSi 기출 AJAX를 사용하되, `--exam-family office`로 시행연도 규칙을 바꾼다.
- 교육청 생활과 윤리/윤리와 사상은 사회탐구 과목 ID를 그대로 사용한다.
- 파일명과 폴더명은 `2026년`처럼 시행연도 기준으로 만든다.
- 통합사회는 공식 루트 확인 전까지 비활성 상태다.
