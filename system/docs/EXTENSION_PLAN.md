# 확장 설계 메모

이 도구는 장기적으로 여러 과목과 여러 시험 주관 기관을 함께 관리할 수 있어야 한다.

## 명칭 원칙

시험의 최종 구분명은 공식 사이트와 공식 파일 제목을 우선한다.

- 평가원/EBSi 기출: `2027학년도 6월 모의평가`, `2027학년도 대학수학능력시험`처럼 학년도 기준 명칭을 사용한다.
- 교육청 학력평가: 실제 시행 연도와 교육청이 쓰는 명칭을 따른다.
- 내부 점검용으로는 `시행연도`, `시행월`, `공식 제목`, `표시 학년도`, `주관기관`을 분리해 저장한다.

형식적으로 평가원 시험은 현 연도에 시행되어도 다음 학년도 대상 시험이다. 예를 들어 2026년에 시행되는 평가원 6월/9월/수능은 `2027학년도`로 관리한다. 반면 교육청 시험은 현재 고등학생의 학력 점검 성격이므로 공식 명칭이 가리키는 현 연도 체계를 따른다.

## 2027학년도처럼 새 시험이 추가될 때

1. UI에서 과목을 선택한다.
2. `다음 학년도까지 확인`을 켠다.
3. `가능여부 확인`으로 공식 사이트에서 현재 받을 수 있는 시험/문서 목록을 표로 확인한다.
4. 공식 사이트에 아직 파일이 없으면 보고서와 표에 `미발견`으로 남긴다.
5. 표에서 원하는 행을 고른 뒤 `선택 다운로드`를 누른다.
6. `누락/재다운로드 보고서`를 만든 뒤 `저장 경로에 반영`을 누르면 선택한 저장 경로로 복사된다.

공식 다운로드 전체 새로고침은 여전히 가능하지만, 신규 시험이 추가되는 시기에는 `가능여부 확인 -> 선택 다운로드` 절차를 우선한다.

## 여러 과목 확장

과목별로 다음 값을 프로필화한다.

- 표시명: `생활과 윤리`, `윤리와 사상`, `통합사회`
- 공식 사이트 과목 ID
- 주관기관: 평가원/EBSi, 교육청
- 원본 점검 폴더
- 저장 경로(복사 목적지)
- 파일명 규칙

현재 EBSi 평가원 어댑터는 `subjects.json`의 과목 목록을 읽는다. 사회탐구 과목은 과목 ID만 있으면 같은 방식으로 가능여부 확인과 선택 다운로드가 가능하다. 통합사회처럼 교육청 또는 새 교육과정의 별도 루트가 필요한 과목은 `enabled=false`로 두고, 별도 어댑터를 붙인다.

## 교육청 확장

교육청 시험은 평가원과 연도 라벨 정책이 다르므로 별도 어댑터로 둔다.

- 평가원 어댑터: `시행연도 + 1 = 학년도`
- 교육청 어댑터: 공식 제목과 시행연도 중심
- 공통 처리: 다운로드 원본 보관, 현황 점검, 누락 보고서, 저장 경로로 복사

내년에 통합사회 관련 작업을 추가할 때도 이 구조를 유지한다. 먼저 공식 다운로드 루트를 식별하고, `subjects.json`에 과목 프로필을 추가한 뒤, 교육청/평가원 중 맞는 어댑터에 연결한다.

## 온라인 루트와 오프라인 루트 분리

온라인 루트는 공식 다운로드 원본을 확보하는 역할만 맡긴다.

- EBSi/평가원/교육청 사이트 접근 확인
- 공식 URL 수집
- 다운로드 원본 보관
- 해시와 파일 형식 검증

오프라인 루트는 사용자의 자료 폴더를 점검하고 정리한다.

- 원본 폴더 스캔
- 크롤링 산출물/원본 PDF/통합본 분류
- 누락 시험 집계
- 공식 다운로드 원본을 저장 경로로 복사

이렇게 나누면 인터넷 연결이 없는 상태에서도 현황 점검과 분류는 계속 가능하다.

## 다운로드 원본 버전 정책

다운로드 출력물은 과목별로 나눈다.

```text
downloads/
  생활과 윤리/
    current/
    legacy/
      YYYYMMDD-HHMMSS/
```

정책:

- `current`는 최신 공식 다운로드 원본이다.
- 같은 파일을 다시 다운로드했는데 해시가 같으면 아무 파일도 늘리지 않고 `unchanged`로 기록한다.
- 같은 파일명인데 해시가 다르면 기존 파일을 `legacy/YYYYMMDD-HHMMSS/` 아래 같은 상대 경로로 이관하고 새 파일을 `current`에 둔다.
- 공식 사이트가 파일을 교체한 경우도 추적 가능하게 매니페스트에 `downloaded`, `unchanged`, `archived`, `skipped`, `failed`를 모두 남긴다.
- 사용자가 보는 `저장 경로`는 실제로는 `current`에서 선택한 대상 폴더로 복사되는 경로다.

이 방식은 매 실행마다 전체를 백업하는 방식보다 저장 공간을 덜 쓰고, 공식 파일이 실제로 바뀐 시점만 추적할 수 있다.
